## Flex 弹性布局

<span>Flex 是 CSS 中最常用的一维布局方案。</span>它擅长处理横向一排或纵向一列的排列、对齐、间距和自适应问题。只要页面中的元素主要沿着一条轴线分布，Flex 通常就是非常自然的选择。

### Flex 适合场景

Flex 适合处理一维方向上的布局，也就是主要关心“横着排”或者“竖着排”的场景。

常见场景：

- 导航栏左右分布
- 按钮组横向排列
- 图标和文字垂直居中
- 卡片内部上下分布
- 表单 label 和输入框对齐
- 列表项内部内容两端对齐
- 弹窗底部按钮靠右排列
- 头像、标题、描述组成的媒体对象

### 基础结构

下面代码表示父元素启用 Flex，子元素会自动成为 flex item，并沿主轴排列。

<CodeWindow
  lang="css"
  code={`
.toolbar {
  display: flex;
  align-items: center;
  gap: 16px;
}
`}
/>

这段代码可以理解为：

- `display: flex` 让父元素变成 flex container
- 直接子元素会成为 flex item
- `align-items: center` 控制交叉轴居中
- `gap: 16px` 控制子元素之间的间距

### 主轴和交叉轴

Flex 里最重要的概念是<span>主轴</span>和<span>交叉轴</span>。

默认情况下：

| 概念 | 默认方向 | 常用控制属性 |
| --- | --- | --- |
| 主轴 | 从左到右 | `justify-content` |
| 交叉轴 | 从上到下 | `align-items` |

如果设置 `flex-direction: column`，主轴会变成从上到下，交叉轴会变成从左到右。

<CodeWindow
  lang="css"
  code={`
.row {
  display: flex;
  flex-direction: row;
}

.column {
  display: flex;
  flex-direction: column;
}
`}
/>

### 常用属性

<span>Flex 常用属性可以分成两类：</span>一类写在父元素上，用来控制整体排列；另一类写在子元素上，用来控制单个项目如何伸缩。

#### display

启用 Flex 布局的入口是 `display: flex`。设置以后，父元素会成为 flex container，直接子元素会成为 flex item。

<CodeWindow
  lang="css"
  code={`
.nav {
  display: flex;
}
`}
/>

如果想让元素自身保持行内特性，也可以使用 `inline-flex`。

<CodeWindow
  lang="css"
  code={`
.tag {
  display: inline-flex;
  align-items: center;
  gap: 6px;
}
`}
/>

#### justify-content

`justify-content` 用来控制子元素在主轴上的分布方式。

<CodeWindow
  lang="css"
  code={`
.nav {
  display: flex;
  justify-content: space-between;
}

.buttonGroup {
  display: flex;
  justify-content: center;
}
`}
/>

常见取值：

| 值 | 效果 |
| --- | --- |
| `flex-start` | 靠主轴起点 |
| `center` | 主轴居中 |
| `flex-end` | 靠主轴终点 |
| `space-between` | 两端贴边，中间平均分 |
| `space-around` | 每个元素两侧都有间距 |
| `space-evenly` | 所有间距完全相等 |

#### align-items

`align-items` 用来控制子元素在交叉轴上的对齐方式。最常见的用途是让图标和文字垂直居中。

<CodeWindow
  lang="css"
  code={`
.menuItem {
  display: flex;
  align-items: center;
  gap: 8px;
}
`}
/>

常见取值：

| 值 | 效果 |
| --- | --- |
| `stretch` | 默认值，拉伸填满交叉轴 |
| `flex-start` | 靠交叉轴起点 |
| `center` | 交叉轴居中 |
| `flex-end` | 靠交叉轴终点 |
| `baseline` | 按文字基线对齐 |

#### gap

`gap` 用来控制 flex item 之间的间距。相比给每个子元素写 `margin-right`，`gap` 更干净，也不会出现最后一个元素多出间距的问题。

<CodeWindow
  lang="css"
  code={`
.buttonGroup {
  display: flex;
  gap: 12px;
}
`}
/>

也可以分别设置行间距和列间距：

<CodeWindow
  lang="css"
  code={`
.tagList {
  display: flex;
  flex-wrap: wrap;
  row-gap: 10px;
  column-gap: 12px;
}
`}
/>

#### flex-direction

`flex-direction` 用来改变主轴方向。

<CodeWindow
  lang="css"
  code={`
.row {
  display: flex;
  flex-direction: row;
}

.card {
  display: flex;
  flex-direction: column;
}
`}
/>

常见取值：

| 值 | 说明 |
| --- | --- |
| `row` | 从左到右排列 |
| `row-reverse` | 从右到左排列 |
| `column` | 从上到下排列 |
| `column-reverse` | 从下到上排列 |

#### flex-wrap

默认情况下，Flex 子元素会尽量挤在一行里。如果内容太多，可以使用 `flex-wrap: wrap` 让它自动换行。

<CodeWindow
  lang="css"
  code={`
.tagList {
  display: flex;
  flex-wrap: wrap;
  gap: 10px;
}
`}
/>

这个属性常用于标签列表、按钮组、筛选条件等内容数量不固定的区域。

#### align-content

`align-content` 只在<span>多行 Flex</span>时生效，用来控制多行内容在交叉轴上的整体分布。

<CodeWindow
  lang="css"
  code={`
.panel {
  display: flex;
  flex-wrap: wrap;
  align-content: flex-start;
  gap: 16px;
}
`}
/>

需要注意：如果只有一行，`align-content` 基本看不到效果，这时应该看 `align-items`。

### 子元素上的属性

父元素负责整体排列，子元素负责自己怎么伸缩。下面这些属性一般写在 flex item 上。

#### flex-grow

`flex-grow` 表示当容器有剩余空间时，子元素是否参与放大。

<CodeWindow
  lang="css"
  code={`
.layout {
  display: flex;
  gap: 20px;
}

.main {
  flex-grow: 1;
}

.aside {
  width: 260px;
}
`}
/>

这里 `.main` 会吃掉剩余空间，`.aside` 保持固定宽度。

#### flex-shrink

`flex-shrink` 表示当空间不足时，子元素是否允许缩小。默认值是 `1`，也就是允许缩小。

<CodeWindow
  lang="css"
  code={`
.avatar {
  flex-shrink: 0;
  width: 48px;
  height: 48px;
}
`}
/>

头像、图标、固定按钮这类元素通常不希望被压扁，可以设置 `flex-shrink: 0`。

#### flex-basis

`flex-basis` 表示子元素在主轴上的初始尺寸。它比 `width` 更贴近 Flex 的语义。

<CodeWindow
  lang="css"
  code={`
.item {
  flex-basis: 240px;
}
`}
/>

#### flex 简写

`flex` 是 `flex-grow`、`flex-shrink`、`flex-basis` 的简写。

<CodeWindow
  lang="css"
  code={`
.main {
  flex: 1;
}

.sidebar {
  flex: 0 0 260px;
}
`}
/>

可以简单理解：

| 写法 | 含义 |
| --- | --- |
| `flex: 1` | 可以放大、可以缩小、自动分配空间 |
| `flex: 0 0 260px` | 不放大、不缩小，固定 260px |
| `flex: none` | 基本不参与弹性伸缩 |

### 典型布局示例

#### 导航栏左右分布

导航栏通常需要左侧放 logo，右侧放菜单或操作按钮。这个场景非常适合 Flex。

<CodeWindow
  lang="css"
  code={`
.nav {
  display: flex;
  align-items: center;
  justify-content: space-between;
  height: 64px;
}

.navLinks {
  display: flex;
  align-items: center;
  gap: 24px;
}
`}
/>

#### 图标和文字对齐

图标和文字不对齐是很常见的问题。使用 Flex 后，只需要让它们在交叉轴居中。

<CodeWindow
  lang="css"
  code={`
.infoItem {
  display: inline-flex;
  align-items: center;
  gap: 8px;
}

.infoItem svg {
  width: 18px;
  height: 18px;
}
`}
/>

#### 卡片内部上下分布

卡片内部经常需要标题在上方，标签或按钮固定贴近底部。可以让卡片使用纵向 Flex，再用 `margin-top: auto` 把底部内容推下去。

<CodeWindow
  lang="css"
  code={`
.card {
  display: flex;
  flex-direction: column;
  min-height: 220px;
  padding: 20px;
}

.cardFooter {
  margin-top: auto;
}
`}
/>

#### 左侧固定，右侧自适应

后台页面、详情页、内容布局经常会用到左侧固定宽度，右侧自适应。

<CodeWindow
  lang="css"
  code={`
.page {
  display: flex;
  gap: 24px;
}

.aside {
  flex: 0 0 260px;
}

.content {
  flex: 1;
  min-width: 0;
}
`}
/>

这里 `.content` 加 `min-width: 0` 很重要，它可以避免右侧内容过长时把布局撑破。

#### 弹窗底部按钮靠右

弹窗底部经常有“取消”和“确认”按钮，通常希望它们靠右排列。

<CodeWindow
  lang="css"
  code={`
.modalFooter {
  display: flex;
  justify-content: flex-end;
  gap: 12px;
}
`}
/>

#### 等宽按钮组

如果希望一组按钮平均分配宽度，可以让每个按钮都使用 `flex: 1`。

<CodeWindow
  lang="css"
  code={`
.actionGroup {
  display: flex;
  gap: 12px;
}

.actionGroup button {
  flex: 1;
}
`}
/>

### 响应式 Flex 布局

Flex 很适合做轻量响应式。比如桌面端左右分布，移动端上下排列。

<CodeWindow
  lang="css"
  code={`
.profile {
  display: flex;
  align-items: center;
  gap: 24px;
}

@media (max-width: 640px) {
  .profile {
    flex-direction: column;
    align-items: flex-start;
  }
}
`}
/>

如果子元素数量不固定，可以配合 `flex-wrap`。

<CodeWindow
  lang="css"
  code={`
.articleTools {
  display: flex;
  flex-wrap: wrap;
  gap: 10px;
}
`}
/>

### 常见问题

#### 子元素内容过长导致布局被撑开

在 Flex 布局中，如果某个子元素内部文本很长，可能会把容器撑开。常见解决方式是给 flex item 设置 `min-width: 0`。

<CodeWindow
  lang="css"
  code={`
.listItem {
  display: flex;
  gap: 12px;
}

.listContent {
  min-width: 0;
}

.title {
  overflow: hidden;
  white-space: nowrap;
  text-overflow: ellipsis;
}
`}
/>

#### flex: 1 为什么有时不等宽

`flex: 1` 会参与剩余空间分配，但如果元素本身内容宽度差异很大，看起来可能仍然不完全等宽。想让它们更稳定，可以配合 `min-width: 0`。

<CodeWindow
  lang="css"
  code={`
.tabs {
  display: flex;
}

.tab {
  flex: 1;
  min-width: 0;
  text-align: center;
}
`}
/>

#### margin 和 gap 怎么选

如果只是控制同级元素之间的间距，优先使用 `gap`。如果是某个元素需要特殊偏移，再考虑 `margin`。

<CodeWindow
  lang="css"
  code={`
.list {
  display: flex;
  gap: 16px;
}

.submitButton {
  margin-left: auto;
}
`}
/>

`margin-left: auto` 在 Flex 中很有用，它可以把某个元素推到另一侧。

#### align-items 没有生效

有时候看起来 `align-items: center` 没有生效，是因为元素高度不一致、内部文本换行，或者你真正想控制的是主轴方向。这时要先判断当前主轴是什么。

<CodeWindow
  lang="css"
  code={`
.row {
  display: flex;
  align-items: center;
}

.column {
  display: flex;
  flex-direction: column;
  justify-content: center;
}
`}
/>

### Flex 和 Grid 怎么选

可以简单这样判断：

| 场景 | 推荐 |
| --- | --- |
| 一行或一列排列 | Flex |
| 同时控制行和列 | Grid |
| 图标文字对齐 | Flex |
| 按钮组、导航栏 | Flex |
| 卡片宫格、图片墙 | Grid |
| 后台复杂区域划分 | Grid |

### 使用建议

- 一维排列优先考虑 Flex
- 图标和文字对齐优先使用 Flex
- 子元素间距优先使用 `gap`
- 内容可能过长时，记得给 flex item 设置 `min-width: 0`
- 固定宽度区域可以使用 `flex: 0 0 宽度`
- 自适应区域可以使用 `flex: 1`
- 同时控制行和列时，更适合使用 Grid

### 小结

Flex 的核心是<span>沿着一条轴线分配空间</span>。当布局问题主要发生在横向或纵向的一维方向上时，Flex 往往是最自然、最轻量的选择。掌握主轴、交叉轴、伸缩规则和常见布局模式后，大多数日常页面结构都可以用 Flex 写得很清晰。
