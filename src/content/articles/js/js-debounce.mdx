## 认识 JS 防抖

> <span>防抖</span>会把连续触发合并成最后一次执行，适合输入搜索、窗口变化和防重复提交。

### 适用场景

- 搜索框停止输入后请求
- resize 结束后计算布局
- 富文本停止输入后保存草稿
- 按钮短时间内只提交一次

### 技术点总览

| 技术点 | 主要作用 |
| --- | --- |
| 基础防抖 | 每次触发时先清除上一次定时器，再重新设置定时器。只有最后一次触发能真正执行。 |
| 输入搜索场景 | 输入框每次输入都会触发 input，如果直接请求接口会太频繁，可以等用户停下来再请求。 |
| 支持取消定时器 | 组件卸载或页面切换时，可以取消还没执行的逻辑，避免异步任务继续修改页面状态。 |

### 分批介绍

#### 基础防抖

每次触发时先清除上一次定时器，再重新设置定时器。只有最后一次触发能真正执行。

<CodeWindow
  lang="js"
  code={`
function debounce(fn, delay = 300) {
  let timer = null

  return function (...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}
`}
/>

#### 输入搜索场景

输入框每次输入都会触发 input，如果直接请求接口会太频繁，可以等用户停下来再请求。

<CodeWindow
  lang="js"
  code={`
const search = debounce((keyword) => {
  fetch('/api/search?keyword=' + keyword)
}, 500)

input.addEventListener('input', (event) => {
  search(event.target.value)
})
`}
/>

#### 支持取消定时器

组件卸载或页面切换时，可以取消还没执行的逻辑，避免异步任务继续修改页面状态。

<CodeWindow
  lang="js"
  code={`
function debounce(fn, delay = 300) {
  let timer = null

  function debounced(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }

  debounced.cancel = () => clearTimeout(timer)
  return debounced
}
`}
/>

### 使用建议

- 防抖强调最后一次。
- 需要连续反馈时不要用防抖。
- 页面销毁时注意清理定时器。
