## 函数组件中的状态逻辑

> <span>Hooks</span> 用来在函数组件中组织状态、副作用、引用和可复用逻辑。

### 适用场景

- 请求数据
- 监听滚动
- 控制弹窗
- 抽取复用逻辑

### 技术点总览

| 技术点 | 主要作用 |
| --- | --- |
| useState 管理状态 | 会影响视图的数据适合放在 useState 中。 |
| useEffect 处理副作用 | 请求、事件监听和定时器都属于副作用。 |
| 自定义 Hook 复用逻辑 | 多个组件都要用的状态逻辑可以抽成自定义 Hook。 |

### 分批介绍

#### useState 管理状态

会影响视图的数据适合放在 <span>useState</span> 中。

<CodeWindow
  lang="jsx"
  code={`
const [count, setCount] = useState(0)

setCount((value) => value + 1)
`}
/>

#### useEffect 处理副作用

请求、事件监听和定时器都属于副作用。

<CodeWindow
  lang="jsx"
  code={`
useEffect(() => {
  const onScroll = () => setTop(window.scrollY)
  window.addEventListener('scroll', onScroll)
  return () => window.removeEventListener('scroll', onScroll)
}, [])
`}
/>

#### 自定义 Hook 复用逻辑

多个组件都要用的状态逻辑可以抽成<span>自定义 Hook</span>。

<CodeWindow
  lang="jsx"
  code={`
function useBoolean(defaultValue = false) {
  const [value, setValue] = useState(defaultValue)
  return [value, () => setValue(true), () => setValue(false)]
}
`}
/>

### 使用建议

- 先把数据来源、状态归属和交互入口写清楚，再考虑抽象。
- 示例代码只保留核心写法，真实业务里还要补充 loading、error、empty 等状态。
- 如果一种写法让代码更绕，优先回到更直观、更容易维护的实现方式。

### 小结

函数组件中的状态逻辑 的重点不是记住某个 API，而是理解它解决的问题、适合的场景以及和其它技术点之间的关系。把每个小知识点拆开看，再组合到真实业务里，代码会更有规律，也更容易维护。
